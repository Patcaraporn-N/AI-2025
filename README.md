## วัตถุประสงค์ของโครงงาน
1.เพื่อพัฒนาระบบคัดกรองอาการป่วยเบื้องต้นผ่านเว็บ
2.เพื่อประยุกต์ใช้ AI ในการวิเคราะห์อาการจากข้อความ
3.เพื่อช่วยให้ผู้ใช้เข้าใจอาการของตนเองเบื้องต้น
4.เพื่อศึกษาแนวทางการพัฒนา Web Application ด้วย Streamlit



## ฟีเจอร์หลัก (Features)
-ผู้ใช้สามารถกรอกอาการเบื้องต้นผ่านหน้าเว็บไซต์ได้
-ระบบวิเคราะห์อาการด้วยแนวคิด AI แบบ Symptom Matching Algorithm
-แสดงโรคที่มีความเป็นไปได้มากที่สุดจากฐานข้อมูลโรค
-แสดงคำแนะนำเบื้องต้นเกี่ยวกับโรคนั้น ๆ
-รองรับการใช้งานผ่าน Web Browser
-ใช้งานง่าย เหมาะสำหรับการคัดกรองอาการเบื้องต้นก่อนพบแพทย์



## การใช้ AI ในระบบ
ระบบนี้ใช้แนวคิดของ Artificial Intelligence (AI) ในการวิเคราะห์อาการของผู้ใช้ โดยใช้วิธี
Symptom Matching Algorithm



## ขั้นตอนการทำงาน
1.ผู้ใช้กรอกอาการเป็นข้อความ
2.ระบบแปลงข้อความเป็นรูปแบบที่สามารถวิเคราะห์ได้
3.เปรียบเทียบอาการกับฐานข้อมูลโรค
4.คำนวณคะแนนความสอดคล้องของอาการ
5.แสดงโรคที่มีความเป็นไปได้สูงสุด
แนวคิดนี้เป็นพื้นฐานของ AI Decision Support System

## การสะท้อนคิด (Reflection)
จากการพัฒนาโครงงานนี้ ทำให้ได้เรียนรู้การการพัฒนาเว็บแอปด้วย python และ streamlit รวมถึงการนำแนวคิด Ai แบบ Rule-based มาช่วยวิเคราะห์อาการจากข้อมูลที่กรอก เพื่อประเมินโรคเบื้องต้น

สิ่งที่ทำได้ดี การสร้างระบบฐานข้อมูลที่เกี่ยวกับโรค เพื่อให้ระบบสามารถนำไปตัดสินและวิเคราะห์โรคที่มีความเกี่ยวข้องกับสิ่งที่กรอกลงไปได้

สิ่งที่ต้องปรับปรุง เนื่องจากมีข้อจำกัดหลายอย่างเช่น วิเคราะห์ได้เฉพาะโรคที่อยู่ในฐานข้อมูลเท่านั้น ไม่สามารถใช้แทนาการวินิจฉัยของแพทย์ ยังไม่ได้ใช้ Machine Learning
ควรเพิ่มฐานข้อมูลอาการ และ โรคให้มากขึ้น และพัฒนาให้ใช้ AI ที่ซับซ้อนได้ขึ้น เช่น Machine Learning เพื่อให้การวิเคราะห์ มีความแม่นยำมากขึ้น

เพิ่มจำนวนโรคและอาการในระบบให้มากขึ้น เพื่อให้วิเคราะห์ได้แม่นยำขึ้น
ปรับปรุงการวิเคราะห์อาการให้รองรับการพิมพ์ที่หลากหลายมากขึ้น
